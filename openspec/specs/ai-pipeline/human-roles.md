# 人类角色重新定义

> 从"执行者"变为"指挥官"。人类只做决策、审批、把关，不做具体执行。

---

## 指挥官三角

| 角色 | 操作 | 产出 |
|------|------|------|
| **决策** | 审批 proposal.md，审批需求边界 | 需求冻结 |
| **审批** | Code Review + 架构合规审查 | 问题清单 |
| **把关** | 最终放行 + 事故升级处理 | 上线许可 |

---

## 实际操作流程

### 1. 决策阶段（人类）
```
人类：输出业务意图 + 架构约束
AI：生成 proposal.md → 人类审批通过 → 冻结需求
```

### 2. 执行阶段（AI）
```
AI：根据 design.md 执行 → 生成 tasks.md → 分步实施
```

### 3. 审批阶段（人类）
```
人类：Code Review → 提出问题 → AI 修复 → 重复直到通过
```

### 4. 把关阶段（人类）
```
人类：最终审查 → 放行或拒绝 → 处理升级事故
```

---

## 为什么重要

- ❌ 人类做执行 → 陷入细节，消耗精力，失去全局视野
- ✅ 人类做决策/审批/把关 → 把控关键节点，保证质量

---

## 触发条件

任何涉及多步骤变更的完整流程。

---

## AI 执行边界

以下操作 **AI 无权自行决定**，必须先获得人类批准：

1. 删除或修改 `openspec/specs/` 下的规范文件
2. 修改 `.clinerules` 执行家法
3. 删除或重命名核心模块
4. 修改 API 端点行为
5. 引入新的外部依赖
6. 任何破坏性数据操作

以下操作 **AI 可以自主决定**：

1. 重构不影响行为的内部实现
2. 添加测试
3. 更新文档（specs 除外）
4. 格式化代码
5. 修复已确认的 Bug
6. 添加日志和错误处理
