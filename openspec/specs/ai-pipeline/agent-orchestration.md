# AI 协作规范

> 最后更新：2026/04/29
> 状态：活跃

---

## AI 执行规则

### 原子化变更

每次变更只解决一个需求：
- 一个 PR / commit / 对话只做一件事
- 单次变更 ≤ 300 行核心逻辑
- 所有自动化操作必须幂等

### 规划前置

涉及 ≥2 个文件或跨模块的变更，必须先生成三层文档：
1. `proposals/` 下的 `proposal.md` — 需求背景与目标
2. `proposals/` 下的 `design.md` — 技术方案
3. `.catpaw/tasks.md` — 实施清单（≤15 项）

### 降级策略

辅助功能失败不阻塞核心功能：
```
1. 优先：主流程执行
2. 降级：本地缓存或 fallback
3. 兜底：静默跳过，不抛错
```

---

## 人类角色

| 阶段 | 人类角色 | 产出 |
|------|---------|------|
| 决策 | 审批 proposal.md | 需求冻结 |
| 执行 | AI 根据 design.md | 分步实施 |
| 审批 | Code Review | 问题清单 |
| 把关 | 最终放行 | 上线许可 |

---

## 触发条件

- 多文件变更（≥2 个）→ 启动规划前置流程
- 外部系统依赖 → 实施降级策略
- 任何代码变更 → 遵守原子化原则
