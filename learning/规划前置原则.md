# 规划前置原则 (Plan-First)

## 核心定义
**严禁直接让 AI 写大段业务代码**。任何涉及多文件或跨模块的变更，必须先生成三层规划文档。

## 三层文档体系

### 1. `proposal.md` — 需求背景与目标
回答"为什么做"和"做什么"：
- 业务背景与驱动因素
- 需求边界与排除项
- 成功标准（可量化）
- 优先级与里程碑

### 2. `design.md` — 技术方案与架构决策
回答"怎么做"：
- 架构拓扑图（数据流/组件关系）
- 技术选型对比与决策理由
- API 接口设计
- 风险点与缓解措施

### 3. `tasks.md` — 实施清单
AI 的"施工单"，控制在 ≤15 项：
- 每项均为原子化任务（可独立 Review）
- 包含验收条件
- 按依赖关系排序

## 为什么重要
- 大段代码一次性输出 → LLM 遗忘上下文 + 幻觉累积
- 三层文档 → 人工审批点前移，避免返工
- 每次变更都有"决策记录"，后期可追溯

## 触发条件
多文件变更（≥2 个文件）或跨模块逻辑。
