# 人类角色重新定义

## 核心定义
从"执行者"变为"指挥官"。人类只做决策、审批、把关，不做具体执行。

## 指挥官三角

| 角色 | 操作 | 产出 |
|------|------|------|
| **决策** | approve proposal.md，审批需求边界 | 需求冻结 |
| **审批** | code review + 架构合规审查 | 问题清单 |
| **把关** | 最终放行 + 事故升级处理 | 上线许可 |

## 实际操作流程

### 1. 决策阶段（我）
```
我：输出业务意图 + 架构约束
AI：生成 proposal.md → 我审批通过 → 冻结需求
```

### 2. 执行阶段（AI）
```
AI：根据 design.md 执行 → 生成 tasks.md → 分步实施
```

### 3. 审批阶段（我）
```
我：code review → 提出问题 → AI 修复 → 重复直到通过
```

### 4. 把关阶段（我）
```
我：最终审查 → 放行或拒绝 → 处理升级事故
```

## 为什么重要
- ❌ 人类做执行 → 陷入细节，消耗精力，失去全局视野
- ✅ 人类做决策/审批/把关 → 把控关键节点，保证质量

## 触发条件
任何涉及多步骤变更的完整流程。
