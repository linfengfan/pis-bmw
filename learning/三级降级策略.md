# 三级降级策略（不阻塞核心路径）

## 核心定义
辅助功能的失败不阻塞核心功能。设计降级链，确保主流程始终可用。

## 三级降级链

```
1. 优先：MCP / 外部工具查询
        ↓ 失败
2. 降级：读取本地缓存或 fallback 配置
        ↓ 失败
3. 兜底：静默跳过，不阻塞主流程
```

## 实施示例

### 示例 1：AI 知识库查询
```typescript
async function queryKnowledge(query: string): Promise<Answer | null> {
  // L1: MCP 外部知识库查询
  try {
    return await mcpService.query(query);
  } catch {
    // L2: 本地 fallback
    return localCache.get(query) ?? null;
  }
  // L3: 静默跳过，不抛错，核心流程继续
}
```

### 示例 2：Analytics 上报
```typescript
async function reportEvent(event: Event) {
  try {
    await analytics.track(event);
  } catch {
    // 降级到本地队列
    localQueue.push(event);
  }
  // 即使 L2 也失败，核心逻辑不受影响
}
```

### 示例 3：非核心 UI 降级
```
推荐模块不可用 → 显示通用热门内容 → 隐藏模块
（用户无感知核心体验下降）
```

## 为什么重要
- AI 能力不稳定（延迟、幻觉、网络抖动）→ 必须有降级链
- 核心功能（用户下单、登录）→ 绝对不能被辅助功能失败影响
- 静默兜底 → 用户体验平滑，不因边缘功能崩溃

## 触发条件
任何依赖外部系统（API、MCP、第三方服务）的场景。
